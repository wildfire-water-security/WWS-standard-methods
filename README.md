# standard-methods
Documentation and scripts related to data collection and QA/QC used across multiple projects

**Why does this repo exist, why not store this info within each project?**
- To ensure we are all processing data the same way across the project for consistency
- All methods are easy to find, located in a single location

## Instructions 
Any standardized scripts, directions, or other files related to collecting data and performing QA/QC on data generated by the project should be stored here so others can easily find and use.

### File Structure 
Within the existing file structure, create a new folder for each unique data type.

**SOPs: Standard Operating Procedures**
  - *Field*: Associated with fieldwork
  - *Lab*: Associated with lab analyses
  - *Modeling*: Associated with modeling analyses

**QAQC-files: Any files related to performing QAQC on data**
  - *Field*: Associated with fieldwork
  - *Lab*: Associated with lab analyses
  - *Modeling*: Associated with modeling analyses

Steps:
1. Navigate to the appropriate folder (e.g., `SOPs/Lab` or `QAQC-files/Field`).
2. Within that folder, create a separate folder for each data type (e.g., `NPOC`, `BPCA`, `DOM`, `Sonde-Data`, etc.).
